Get started with WAMP by choosing client libraries and a router:

  • To write application components in your favorite language, you will need a WAMP client library
  • To actually run your application consisting of components, you will need a WAMP router


Notes
- You can choose your WAMP router independently of the programming language you want to write your components in since routers do not run application code directly.
- Want your WAMP implementation listed here? Please join the WAMP group and introduce your project.


WAMP Libraries

The following table lists WAMP compliant client library implementations. For each implementation, an "x" marks a WAMP role supported by the implementation. The last column marks principle support for features from WAMP Advanced Profile.

Library Language Description
akka-wamp Scala, Java 8 WAMP implementation written in Scala with Akka HTTP, usable from Java 8.
AngularWAMP JavaScript AngularWAMP is an AngularJS wrapper for Autobahn|JS that provides deeper integration for AngularJS apps.
Autobahn|Cpp C++ C++ 11 implementation using Futures, Continuations and Lambdas. Runs on top of Boost and ASIO.
Autobahn|Java Java WAMP Client library for Java and Android applications.
Autobahn|JS JavaScript WAMP Client library for both browsers and NodeJS using Promises (when, Q, jQuery or ES6).
Autobahn|Python Python WAMP Client library and WebSocket implementation, runs under Python 2 and 3, on Twisted and asyncio.
awre Erlang awre is a WAMP V2 implementation in Erlang with client roles.
backbone.WAMP JavaScript Allows two-way synchronisation of Backbone models & collections between frontends and backends. Uses AutobahnJS
connectanum Java A Java WAMP implementation based on Netty. Both Client and Router roles.
CppWAMP C++11 CppWAMP is a WAMP V2 implementation in C++11
haskell-wamp Haskell A very experimental Haskell implementation of WAMP Basic Profile.
jawampa Java A Java WAMP implementation based on Netty and RxJava. Both Client and Router roles.
Loowy Lua LUA WAMP client implementation on top of lua-websockets and lib-ev
MDWamp Objective-C MDWamp is WAMP client library for iOS apps that uses SocketRocket as underlying WebSocket implementation.
Minion PHP A simple and flexible WAMP v2 client and command line tool with Laravel support, based on Thruway.
Nexus Go full-feature WAMP client written in Go (also: router)
p5-Net-WAMP Perl A WAMP implementation in Perl - see the docs for details.
ruby_wamp_client Ruby A WAMP client for Ruby
rx.wamp JavaScript A Reactive (RxJS) wrapper library for WAMP, using AutobahnJS and running in the browser and NodeJS.
Thruway PHP Thruway is a WAMP library built in PHP that provides both Client and Router roles.
Turnpike Go A Go implementation of WAMP V2
Spell Elixir/Erlang Spell is a WAMP library built using Elixir, running on the Erlang VM.
Swamp Swift The WAMP WebSocket subprotocol implemented purely in Swift using Starscream & SwiftyJSON.
wampcc C++ A C++ WAMP library that aims to depend only on C libraries, making it easier to work and build on a range of platforms. (Includes basic router)
WAMP_POCO C++ A fork of AutobahnCpp that is using POCO instead of Boost. Uses JSON + WebSocket, with ticket and WAMP-CRA authentication available.
wamped C++ Intended for running on ARM mbed OS. Made for experimental purposes.
WampSharp C# C# implementation of WAMP: both client and router roles, both JSON and MsgPack support.
wampy Python Python implementation not requiring Twisted or asyncio, enabling use within classic blocking Python applications.
wampy.js JavaScript WAMP Client library for browsers using callbacks. See here for comparison to Autobahn|JS.
kraftfahrstrasse TypeScript Modern WAMP Client library for browsers and node utilizing native ES6+.

Router Description
akka-wamp A Scala implementation of a basic WAMP router.
Bonefish WAMP router based on C++11 and Boost.Asio. Also usable as a library, Apache 2.0 licensed.
connectanum A Java WAMP implementation based on Netty. Both Client and Router roles.
Crossbar.io Crossbar.io is an advanced, open-source WAMP router made and supported by Crossbar.io GmbH (creators of WAMP and Autobahn).
Erwa Erwa is a WAMP V2 implementation in Erlang with Router roles.
jawampa A Java WAMP implementation based on Netty and RxJava. Both Client and Router roles.
Nexus Router implementation for Go. (also includes a client implementation)
Nightlife-Rabbit Router implementation for NodeJS, MIT licensed.

turnpike turnpike is a WAMP v2 router implemented in Go.
wamp2spring A Java implementation built on top of the Srping 5 WebSocket support.
wampcc A C++ WAMP library that aims to depend only on C libraries, making it easier to work and build on a range of platforms. (Includes client as well)

WampSharp C# router implementation (client roles are also supported). JSON and MsgPack support.
